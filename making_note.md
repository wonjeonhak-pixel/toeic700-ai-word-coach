# 制作メモ — QuietStep（TOEIC700 AI英単語コーチ）

## このアプリで本当に解決したかったこと

私は TOEIC800 点保持者です。「TOEIC700 を目指す社会人向けの AI 英単語アプリ」というテーマを見たとき、自分が 700 帯を越えるときに体感した「壁」をまず思い出しました。

700 帯では Reading / Listening それぞれ 300〜400 点程度が必要になります。このレベルの本当の難しさは語彙数そのものではなく、**「単語を知っている」と「単語の意味が 1 秒以内に浮かぶ」の間にある段差** にあります。「思い出す」モードでは試験本番のリーディングは時間切れになり、リスニングは音声が次のチャンクへ流れていってしまいます。

加えて、業務でも試験でも、単語が **単体で耳に入る場面はほとんどない**。常に文の中で、音声として、しかも実務文脈で現れます。だから単語カードで意味を当てる学習だけでは、いざ使う/聞く場面で「文の流れに乗れない」のが私自身の経験でした。

## 設計に落とした主な判断

**QuietStep** では、以下を意識的に組み込みました。

- 1 セッションを **10 問 / 約 3 分** に固定し、通勤・昼休みでも完結する単位にする
- 毎問のフィードバックに **`business_example`（英語12語以内の実務例文）+ 🔊 音声再生** を必ずセットで返す（瞬時理解とリスニングの両方の入口になるように）
- 不正解時は AI 誤答分析で **`mistake_type` 5択 + ユーザーの選択肢を名指しで対比** し、「意味の取り違えです」のような無情報フィードバックを構造的に排除
- 10 問終了後の総括では、観測事実主義（前半/後半正答数・mistake_type 分布・弱点 scene）だけを根拠に書かせ、hallucination を抑制
- `learner_type`（〜タイプ）という性格診断風の前向きキャッチコピーで「今日の自分」を 1 行ラベリング、streak やバッジに頼らずに再訪動機を作る

## quiet UX を選んだ理由

過去に自分も英単語アプリを 3 日でやめた経験があります。そのときの感覚は「圏外がストレスだった」ではなく「**毎日やらなきゃという圧そのもの**」が嫌だったというものでした。連続学習日数が途切れた瞬間に開かなくなるという、ゲーミフィケーション特有のサンクコスト的な脱落です。

そのため **QuietStep** では、

- streak / バッジ / ランキング / プッシュ通知を**意図的に置かない**
- 文言から「毎日」「連続」「必ず続けましょう」「達成」を排除する
- 結果画面に「次回の重点単語」最大 3 語だけを静かに残し、再訪時の戻り先を示す

という方針で「**5 分で終われる**」「**また戻ってきやすい**」「**毎日やらなきゃと感じさせない**」という静かな前進感に振り切りました。実装途中でも文言の圧を点検し、ホーム sub の「通勤30分・昼休み15分で前進」を「今日の10問だけ、静かに前進」へ、fallback growth の「積み上げを続けましょう」を「今日の前進が積み上がっています」へ、と命令形を観測表現に置換しています。

ライトテーマ（オフホワイト基調・極薄シャドウ）への切り替えも、夜の通勤電車で開く想定の負担を下げるためで、Apple Books や Notion のような「静かな上質感」を参考にしました。

## AI との共創プロセスについて

「AI 駆動開発」と聞くと「ざっくり伝えて生成させる」イメージがあるかもしれませんが、本アプリでは逆に、**戦略・PRD・バックログを先に言語化** してから Claude Code に渡すアプローチを取りました。

- `specs/strategy.md` — コンセプト・ターゲット・解決する3課題・提供5価値・非ゴール
- `specs/prd.md` — 13 項目の要求仕様と受け入れ基準
- `specs/backlog.md` — AI-DLC 的に小さな単位へ分解した実装タスク（全完了）
- `submission_prompts.md` — Claude API へ実際に渡しているプロンプト全文と設計意図

仕様を先に文書化することで、プロンプト調整の判定軸が安定し、「実 Claude テストで簡体字が混入 → 禁止ルールを追加」「lump-labeling を観測 → 一括ラベル禁止ルールを追加」のような **観測駆動の改善履歴** を残せました。AI と認識合わせをし続けるためのドキュメントが、結果として審査員にも読みやすい提出物になっています。

## 一言でまとめると

「単語を **思い出す** から、**すぐ分かる** へ」。

その入口を、quiet UX で疲れにくく、AI 誤答分析で具体的に、例文＋音声で実用的に — 3 分のセッションに圧縮したのが本アプリです。
