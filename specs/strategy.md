# Strategy — TOEIC700 AI英単語コーチ

本ドキュメントは、社内コンペ第8回提出作品のプロダクト戦略を整理したものです。実装に先立って Claude Code に対して提示し、共通の認識として参照しました。

---

## アプリコンセプト

**「なぜ間違えたか」が分かり、TOEIC700への前進を実感できるAI英単語コーチ。**

単語の暗記アプリではなく、「ビジネス英語として使える感覚」と「自分の弱点に対する解像度」を1セッション3分で返す、AI誤答分析駆動の学習体験を提供します。

---

## ターゲット

通勤30分・昼休み15分しか学習時間が取れない **32歳の社会人**。

- TOEIC700点を目指しているが、まとまった学習時間が取れない
- 過去に英単語アプリを3日でやめた経験がある
- 「今日これをやれば前進している」という実感を求めている
- 強いゲーミフィケーション（連続学習ノルマ・ランキング）にむしろ疲れる

---

## 解決する課題

1. **覚えた単語を使える感覚がない**
   単語と日本語訳の暗記だけでは、メール・会議・報告でその語が口から出てこない。
2. **間違えた理由が分からない**
   「なんとなく違う」で次の問題に進んでしまい、同じ間違いを繰り返す。
3. **続けても前進感がない**
   正答数しか出ないアプリでは、点数の上下に一喜一憂するだけで成長の手応えが残らない。

---

## 提供価値

1. **AI誤答分析**
   不正解時、Claude API がユーザーの選んだ訳と正解の差を具体的に対比し、`mistake_type`（5択）・原因・覚え方・励ましを1セットで返す。「なぜ間違えたか」が毎問言語化される。
2. **`business_example` による実務文脈化**
   毎問のAIフィードバックに、その単語をメール・会議・報告で使う英語例文を1文添える。意味だけでなく「使い方」が貼り付く。
3. **`learner_type` による自分専用感**
   10問終了後、観測事実（`mistake_type` 分布・弱点 scene）を根拠に「〜すると伸びるタイプ」という前向きな性格診断風キャッチコピーを1行返す。「今日の自分」がラベリングされる。
4. **次回の重点単語による戻りやすさ**
   結果画面で、不正解語のうち最大3語を `word + meaning` の小タグで提示。次に開いたとき「ここから戻ればいい」が一目で分かる。
5. **quiet UX による疲れにくい学習体験**
   ダーク基調・1画面1主役・短いAIフィードバック・命令形を避けた文言。アプリを開くこと自体が負担にならない設計。

---

## 設計思想 — quiet UX

強いゲーミフィケーションや連続学習の圧ではなく、以下の3点を重視します。

- **「5分で終われる」** — 1セッション10問・約3分で完結。続きを残さない。
- **「また戻ってきやすい」** — 結果画面に次の重点単語を残し、再訪時の心理的ハードルを下げる。
- **「毎日やらなきゃと感じさせない」** — streak / バッジ / ランキング / 派手な達成演出を意図的に置かない。文言から「続けましょう」「毎日」「達成」を排除する。

ゲーミフィケーションで強制的に開かせるのではなく、「また開きたくなる静かな前進感」で再訪を生むことを目指します。

---

## 非ゴール

意図的に実装しないこと（quiet UX を保つために重要）。

- ログイン / ユーザーアカウント
- 連続学習日数（streak）の表示
- バッジ / ランキング / 称号
- プッシュ通知 / リマインダー
- スコア競争 / 他ユーザー比較
- 詳細な学習履歴ダッシュボード

これらは「毎日やらなきゃ」という圧を生み、quiet UX を壊すため除外しています。

---

## 成功の判定

- 審査員が画面遷移1回分（ホーム → 10問 → 結果）で「なぜ間違えたか」「次に何をやればいいか」を体感できる
- 不正解語に対するAIの説明が、汎用的な「意味の取り違えです」ではなく、ユーザーの選択肢を名指しで説明できている
- 結果画面で `learner_type` と「次回の重点単語」を見て、再訪したくなる
